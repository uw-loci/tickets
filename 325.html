Description

John Alexander writes:

"In my slidebook files, every single image usually has the same image name. Currently, the bio-formats reader just imports the image name as '.sld name - image name' (or something like that). I should also note that this can often create excessively long image names since I use very descriptive .sld file names as well as descriptive image names, so combining the two makes for a very long image name - perhaps prefixing with the .sld filename could be optional? Perhaps there could be a 'sequentially number images' option as well? I was also thinking of asking for a 'add channel name/number?' option as well, but since that information is now in the metadata I could just as easily write a macro to do it myself.

"Of course having all these options can get out of hand quickly. Perhaps having a string formatter to parse that indicates the desired information in the image name might be an easier rout to go. Something like '%f-%n-%c-%C' would be parsed as '.sld filename'-'image name'-'channel number'-'channel name' ... just an idea to simplify input options yet still provide a lot of customization ability to the end user."
