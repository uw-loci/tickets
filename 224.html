Description

From WSR:

One thing that would be nice would be to have the metadata displayed when the user presses "i" (Image>Show Info). This can be done by storing it as the "Info" image property, which is what ImageJ does for DICOM and FITS metadata. Macros can access the "Info" property using the getImageInfo() macro function. Storing the metadata as key1=value1/nkey2=value2, etc. string would make it easier for plugins to decode.
