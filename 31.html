Save/restore state feature is broken

Reported by: curtis Owned by: curtis
Priority: critical Milestone: visbio3.50
Component: visbio Severity: serious
Keywords: Cc:
Blocked By: Blocking:

Description (last modified by curtis)

In theory, VisBio allows the state to be saved to an XML file which can later be restored to return to the exact point where the software was previously. In practice, exceptions are thrown in many cases. Things seem to work with datasets mapped to displays, but adding (e.g.) an overlays transform to the mix causes problems.

The state logic needs to be enhanced anyway to add support for more efficient undo (see ticket #30), so perhaps these bugs can be addressed as part of the overhaul.
