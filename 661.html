Post-acquisition Forward workflow

Reported by: amjohnson27 Owned by: amjohnson27
Priority: minor Milestone:
Component: forward Severity: non-issue
Keywords: Cc:
Blocked By: Blocking:

Description

Currently, the workflow that experimenters use to get data into Forward happens during acquisition (mostly in WiscScan). It would be useful to have a way for experimenters to decide to submit datasets to Forward later in the data lifecycle. This would require either a standalone tool or modifications to an existing tool (e.g., OMERO) that would allow researchers to opt in to sharing data with Forward and to add descriptive information (title, description, keywords). This tool would also need to execute the OME-XML to METS/MODS XSLT stylesheet in order to create the records Forward needs. It would also need to save data to a location where the cron job that generates thumbnails and uploads the METS files and images to Forward is running.
