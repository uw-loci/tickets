Additional constructors

Reported by: curtis Owned by: melissa
Priority: major Milestone:
Component: notes Severity:
Keywords: Cc:
Blocked By: Blocking:

Description

The existing constructor requires a template file and an OME-XML source file as arguments. This situation is not always appropriate. For example, the 4D Data Browser needs to construct a window by passing in an org.openmicroscopy.xml.OMENode object rather than an OME-XML source file on disk, in order to properly display metadata. It would be good to provide a variety of constructors for various circumstances, including some that use a default template (i.e., no template file argument).
