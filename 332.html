Report field count for multifield data

Reported by: melissa Owned by: melissa
Priority: major Milestone: bio-formats-4.0
Component: bio-formats Severity: serious
Keywords: Cc: andrew.vaughan@…
Blocked By: Blocking:

Description

All readers that support multiple fields (Fluoview/Andor TIFF, InCell, Flex, etc.) should use the sub-dimension logic to report the field count. We will need to add a new "FormatTools.MULTIFIELD" constant.

Change History

comment:1 Changed 11 years ago by melissa

  • Milestone set to bio-formats-4.0.0

comment:2 Changed 11 years ago by melissa

At ASCB, we tentatively agreed that multiple fields should be expressed as multiple series (instead of using the sub-C logic). For example, the Andor TIFF sample MF-2CH-Z-T.tif would have 2 channels, 4 Z sections, 5 timepoints, and 5 series.

comment:3 Changed 11 years ago by melissa

  • Cc andrew.vaughan@… added

(In [4736]) FluoviewReader now expresses multifield data as multiple series; see #332.

comment:6 Changed 10 years ago by melissa

  • Status changed from new to closed

(In [4805]) Split each field into its' own series. Closes #332.
